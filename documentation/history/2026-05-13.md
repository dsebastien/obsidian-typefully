# 2026-05-13 — Obsidian community review fixes

Addressed the warnings listed on https://community.obsidian.md/account/plugins/typefully for review of release 3.2.1. Same pattern as the parallel pass on `obsidian-dataview-serializer` and `obsidian-update-time`. Reference checklist at `~/wks/fix-obsidian-review.md`.

## Manifest

- `manifest.json` description rewritten so it does not start with the plugin name and ends with a period: `"Typefully integration. Publish social media posts with ease"` → `"Publish social media posts with ease using the Typefully integration."`. Mirrored the same description into `package.json`.
- `manifest.json` `minAppVersion` bumped from `1.4.0` to `1.7.2` so the use of `Workspace.revealLeaf` (1.7.2) and `Vault.getFileByPath` (1.5.7) no longer triggers warnings.

## Source code

- `eslint.config.ts`: replaced the deprecated `tseslint.config(...)` wrapper with `defineConfig([...])` imported from `eslint/config`; added `activeWindow` and `activeDocument` to the readonly globals so they don't trip `no-undef` after the timer fixes.
- `src/app/api/typefully-api-client.ts` (line 301), `src/app/plugin.ts` (273), `src/app/views/pages/draft-edit-page.ts` (249, 271), `src/app/views/pages/drafts-list-page.ts` (93, 114, 135): replaced `setTimeout(...)` with `activeWindow.setTimeout(...)` for popout-window compatibility.
- `src/app/settings/settings-tab.ts` (line 415): `tagsContainer.createEl('span', {...})` → `tagsContainer.createSpan({...})`.
- `src/app/views/pages/analytics-page.ts` (line 337): `filters.createEl('span', {...})` → `filters.createSpan({...})`.
- `src/app/views/pages/queue-schedule-page.ts` (line 80): `container.createEl('div', {...})` → `container.createDiv({...})`.

## Styles

- `src/styles.src.css`: removed `width: 100% !important;` on `.typefully-modal-input-wide` and bumped specificity via a doubled-class selector. Tightened the `.typefully-settings-tags` margin shorthand from `8px 0 12px 0` to `8px 0 12px` to satisfy `shorthand-property-no-redundant-values`.

## CI / release

- `.github/workflows/ci.yml`, `.github/workflows/release.yml`: replaced `bun-version: latest` with `bun-version-file: package.json` so CI uses the pinned `packageManager` and release builds become reproducible. `packageManager` bumped to `bun@1.3.14`. Attestation step already configured.
- The "release contains additional files: obsidian-typefully.zip" warning can't be fixed retroactively for 3.2.1, but the current `release.yml` only uploads `main.js`, `manifest.json`, and `styles.css`, so the next release will clear it.
- `lint-staged` warning already resolved by the prior `chore(all): removed husky` commit (`51cfb5e`); no further action needed.
